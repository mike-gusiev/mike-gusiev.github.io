Conditional Blocks

  1. Expressions should not be too complex. Maximum number of allowed conditional operators in an expression - 3! Example 
    
//bad
if (x === 3 && a === 3 || b === 2 || c === 6) {
    // ...
}

//good
if (x === 3 && a === 3 || b === 2) {
    // ...
}
  2. Literal boolean values should not be used in condition expressions Example 
    
//bad
if (booleanVariable === true) {
    //...
}
if (booleanVariable !== true) {
    //...
}
if (booleanVariable || false) {
    //...
}
doSomething(!false);


//good
if (booleanVariable) {
    //...
}
if (!booleanVariable) {
    //...
}
if (booleanVariable) {
    //...
}
doSomething(true);
  3. Control flow statements "if", "for", "while", "switch" and "try" should not be nested too deeply. Maximum nesting level - 3.
  4. If you're using multi-line blocks with if and else, put else on the same line as your if block's closing brace. Example 
    
// bad
if (test) {
    thing1();
    thing2();
}
else {
    thing3();
}

// good
if (test) {
    thing1();
    thing2();
} else {
    thing3();
}
  5. "===" and "!==" should be used instead of "==" and "!="
  6. Use shortcuts. Example 
    
// bad
if (name !== '') {
    // ...stuff...
}

// good
if (name) {
    // ...stuff...
}

// bad
if (collection.length > 0) {
    // ...stuff...
}

// good
if (collection.length) {
    // ...stuff...
}
  7. "switch" statements should have at least 3 "case" clauses. Otherwise, use "if" statement.
  8. "switch" statements should end with a "default" clause
  9. Related "if/else if" statements and "cases" in a "switch" should not have the same condition Example 
    
//bad
if (param === 1) {
    openWindow();
} else if (param === 2) {
    closeWindow();
} else if (param === 1) {
    moveWindowToTheBackground();
}
  10. Two branches in the same conditional structure should not have exactly the same implementation Example 
    
//bad
if (param === 1) {
    doSomething();
} else if (param === 2) {
    closeWindow();
} else if (param === 3) {
    doSomething();
}

//bad
switch (i) {
    case 1: {
        doSomething();
        break;
    }
    case 2: {
        doSomethingDifferent();
        break;
    }
    case 3: {
        doSomething();
        break;
    }
    default: {
        doTheRest();
    }
}
  11. Collapsible "if" statements should be merged Example 
    
//bad
if (x !== undefined) {
    if (x === 2) {
        // ...
    }
}

//good
if (x !== undefined && x === 2) {
    // ...
}
  12. When you have if-else statement, don't use negative condition as first block Example 
    
//bad
if (!hasItems) {
    // negative block
} else {
    // positive block
}

//good
if (hasItems) {
    // positive block
} else {
    // negative block
}
  13. You can use ternary operator, but only when you need to return a statement. Don't use it instead of if-else Example 
    
// bad
hasItems ?
    this.updateItems():
    this.getItems();

let title;
if (hasItems) {
    title = 'has items';
} else {
    title = 'no items';
}

// good
if (hasItems) {
    this.updateItems():
} else {
    this.getItems
}
const title = hasItems ? 'has items' : 'no items';
  14. You should always use curly braces in if statements. curly braces can be omitted only when you have a short, 1-line if. Example 
    
// bad
if (hasItems) this.updateItems():
else this.getItems();

// good
if (hasItems) {
    this.updateItems():
} else {
    this.getItems
}

// good
if (hasItems) return this.items;
if (hasItems) {
    return this.items;
}
  15. You should refactor complex if statements, to make it more readable Example 
    
// bad
if (this.items.length && this.state.isVisible && !(this.props.isLoading || this.items === null)) {
    this.showHint();
}

// good
const hasItems = this.items.length && this.state.isVisible;
const isLoading = this.props.isLoading || this.items === null;
if (hasItems && !isLoading) {
    this.showHint();
}
