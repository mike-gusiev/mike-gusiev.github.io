Variables Declarations
Best practices Spaces and code formating Functions Declarations Conditional Blocks Classes and Modules
  1. Variables should start with a lower case letter and proceed with camel case notation. (eg: someAmazingVariable)
  2. Avoid single letter names. Be descriptive with your naming of variables and functions.
  3. const should have priority over let. Always declare your variable with const, and when it starts to mutate - rename to let. Use const even for mutated objects, unless it were re-declared.
  4. Don't create too much variables, try to avoid new variables whenever it possible. You can also extract part of your code to a new variable when you need to fit your line into 120 symbols.
  5. Don't use "future reserved words" as name of the variables and function Example 
    
await, class, const, enum, exports, extends, implements, import, interface, let, package, private, protected, public,
static, super, yield
  6. Variables and functions should not be re-declared Example 
    
//bad
let a = 3;
...
let a = 10;
    
//good
let a = 3;
...
let b = 10;
  7. Variables should not be shadowed Example 
    
//bad
let a = 3;
if (arr.length) {
    let a = 10;
}
    
//good
let a = 3;
if (arr.length) {
    let b = 10;
}
  8. Use the literal syntax for object and array creation Example 
    
// bad
const obj = new Object();
const arr = new Array();
    
// good
const obj = {};
const arr = [];
  9. Use single quotes for strings or string templates Example 
    
//bad
const str = "This is string";
    
//good
const str = 'This is string';
const str2 = `${str}. Second string`;
  10. Variables should always be declared before they are used Example 
    
//bad
$block = this.$element.find('.block');
    
//good
const $block = this.$element.find('.block');
    
//good
let $block;
...
$block = this.$element.find('.block');
  11. Declare unassigned variables last Example 
    
// bad
let i;
let items = getItems();
let dragonball;
let goSportsTeam = true;
let len;
    
// good
let items = getItems();
let goSportsTeam = true;
let dragonball;
let length;
let i;
  12. All calculated values which assigned to a variables should be used. Unused variables (Dead Stores) should be removed Example 
    
//bad
getData: () => {
    const items = this.container.find('.item');
    const data = this.getValues();
    return data;
}
    
//good
getData: () => {
    const data = this.getValues();
    return data;
}
  13. Use of "var" is forbidden. Use "let" or "const" instead. Example 
    
// bad
var snack = 'Meow Mix';
function getFood(food) {
    if (food) {
        var snack = 'Friskies';
        return snack;
    }
    return snack;
}
console.log(getFood(false)); // undefined
    
// good. it allows to avoid hoisting
let snack = 'Meow Mix';
function getFood(food) {
    if (food) {
        let snack = 'Friskies';
        return snack;
    }
    return snack;
}
console.log(getFood(false)); // 'Meow Mix'
  14. "const" must be used for fixed variables Example 
    
// bad
var str = 'Hello world!';
console.log(str);
    
// bad
let str = 'Hello world!';
console.log(str);
    
// good. immutable variables must always be constants!
const str = 'Hello world!';
console.log(str);
  15. Use destructuring instead of direct assignment Example 
    
// bad
var arr = [1, 2, 3];
var a = arr[0];
var b = arr[1];
var c = arr[2];
    
// good
let [a, b, c] = [1, 2, 3];
    
// bad
var luke = { occupation: 'jedi', father: 'anakin' };
var occupation = luke.occupation; // 'jedi'
var father = luke.father; // 'anakin'
    
// good
let luke = { occupation: 'jedi', father: 'anakin' };
let {occupation, father} = luke;
  16. Use ES6 built-in data-structures Example 
    
// Sets
const s = new Set();
s.add("hello").add("goodbye").add("hello");
s.size === 2;
s.has("hello") === true;

// Maps
const m = new Map();
m.set("hello", 42);
m.set(s, 34);
m.get(s) == 34;

// Weak Maps
const wm = new WeakMap();
wm.set(s, { extra: 42 });
wm.size === undefined

// Weak Sets
const ws = new WeakSet();
ws.add({ data: 42 });
  17. Use template Literals for complex strings Example 
    
// bad
var text = "This string contains \"double quotes\" which are escaped.";
var text2 = 'My cat is named ' + name + ' and is ' + age + ' years old.';

// good
let text = `This string contains "double quotes" which don't need to be escaped anymore.`;
let text2 = `My cat is named ${name} and is ${age} years old.`;

let today = new Date();
let text3 = `The time and date is ${today.toLocaleString()}`;
  18. Use "String.includes" instead of "String.indexOf" Example 
    
// bad
var string = 'food';
var substring = 'foo';
console.log(string.indexOf(substring) > -1);

// good
const string = 'food';
const substring = 'foo';
console.log(string.includes(substring));
  19. Use "String.repeat" Example 
    
// bad
function repeat(string, count) {
    var strings = [];
    while(strings.length < count) {
        strings.push(string);
    }
    return strings.join('');
}
var text = repeat('meow', 3);

// good
let text = 'meow'.repeat(3);
