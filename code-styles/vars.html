Variables Declarations
Best practices Spaces and code formating Functions Declarations Conditional Blocks Classes and Modules
  1. Variables should start with a lower case letter and proceed with camel case notation. (eg: someAmazingVariable)
  2. Avoid single letter names. Be descriptive with your naming of variables and functions.
  3. Don't use "future reserved words" as name of the variables and function Example 
    
await, class, const, enum, exports, extends, implements, import, interface, let, package, private, protected, public,
static, super, yield
  4. Variables and functions should not be re-declared Example 
    
//bad
let a = 3;
...
let a = 10;
    
//good
let a = 3;
...
let b = 10;
  5. Variables should not be shadowed Example 
    
//bad
let a = 3;
if (arr.length) {
    let a = 10;
}
    
//good
let a = 3;
if (arr.length) {
    let b = 10;
}
  6. Use the literal syntax for object and array creation Example 
    
// bad
const obj = new Object();
const arr = new Array();
    
// good
const obj = {};
const arr = [];
  7. Use single quotes for strings Example 
    
//bad
const str = "This is string";
    
//good
const str = 'This is string';
  8. Variables should always be declared before they are used Example 
    
//bad
$block = this.$element.find('.block');
    
//good
const $block = this.$element.find('.block');
    
//good
let $block;
...
$block = this.$element.find('.block');
  9. Declare unassigned variables last Example 
    
// bad
let i;
let items = getItems();
let dragonball;
let goSportsTeam = true;
let len;
    
// good
let items = getItems();
let goSportsTeam = true;
let dragonball;
let length;
let i;
  10. All calculated values which assigned to a variables should be used. Unused variables (Dead Stores) should be removed Example 
    
//bad
getData: () => {
    const items = this.container.find('.item');
    const data = this.getValues();
    return data;
}
    
//good
getData: () => {
    const data = this.getValues();
    return data;
}
  11. Use of "var" is forbidden. Use "let" or "const" instead. Example 
    
// bad
var snack = 'Meow Mix';
function getFood(food) {
    if (food) {
        var snack = 'Friskies';
        return snack;
    }
    return snack;
}
console.log(getFood(false)); // undefined
    
// good. it allows to avoid hoisting
let snack = 'Meow Mix';
function getFood(food) {
    if (food) {
        let snack = 'Friskies';
        return snack;
    }
    return snack;
}
console.log(getFood(false)); // 'Meow Mix'
  12. "const" must be used for fixed variables Example 
    
// bad
var str = 'Hello world!';
console.log(str);
    
// bad
let str = 'Hello world!';
console.log(str);
    
// good. immutable variables must always be constants!
const str = 'Hello world!';
console.log(str);
  13. Use destructuring instead of direct assignment Example 
    
// bad
var arr = [1, 2, 3];
var a = arr[0];
var b = arr[1];
var c = arr[2];
    
// good
let [a, b, c] = [1, 2, 3];
    
// bad
var luke = { occupation: 'jedi', father: 'anakin' };
var occupation = luke.occupation; // 'jedi'
var father = luke.father; // 'anakin'
    
// good
let luke = { occupation: 'jedi', father: 'anakin' };
let {occupation, father} = luke;
