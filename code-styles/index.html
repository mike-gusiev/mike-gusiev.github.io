Best Practices
Code Styling Variables Declarations Functions Declarations Conditional Blocks Classes and Modules
  1. Whenever you see a bad code, you have to refactor it! You should always leave a better code, then it was before you.
  2. Always write (and refactor) code into ES6 styles. ES5 is already obsolete! You may use ES5 only for native modules, which are not being compiled by WebPack, Gulp, etc.
    Example 
    
//bad
var x = 3;
function myFunc() {
    return 10;
}

//good
let x = 3;
const myFunc = () => 10;
