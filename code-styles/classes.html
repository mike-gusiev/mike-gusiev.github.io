Classes and Modules
Best practices Spaces and code formating Variables Declarations Functions Declarations Conditional Blocks
  1. Class names should start with an upper case letter and use camel case notation. (Footer, WeatherHub, BaseModule)
  2. Each function should be separated by empty line.
  3. Don't use reserved words as keys. It won't work in IE8. Use readable synonyms in place of reserved words. Example 
    
// bad
const superman = {
    class: 'alien',
    default: { clark: 'kent' },
    private: true
};
    
// bad
const superman = {
    klass: 'alien'
};
    
// good
const superman = {
    defaults: { clark: 'kent' },
    hidden: true,
    type: 'alien'
};
  4. When saving a reference to this use self. Example 
    
//bad
const _this = this;
    
//bad
const that = this;
    
//good
const self = this;
  5. The names of model properties should not contains spaces and quotes Example 
    
//bad
defaults: {
    'first name': 'Bob',
    'birth date': new Date()
}

//good
defaults: {
    firstName: 'Bob',
    birthDate: new Date()
}
