Spaces and code formating
Best practices Variables Declarations Functions Declarations Conditional Blocks Classes and Modules
  1. Use 4 spaces instead of tabs. Configure your IDE to always use 4 spaces in every language. PhpStorm uses 2 tabs in some languages (CSS, Less, JSON), make sure that you've set 4 spaces for that languages (Preferences - Editor - Code Style).
  2. Place 1 space before the leading brace Example 
    
// bad
function test(){
    console.log('test');
}
const test = () =>{
    console.log('test');
}
dog.set('attr',{
    age: '1 year',
    breed: 'Bernese Mountain Dog'
});

// good
function test() {
    console.log('test');
}
const test = () => {
    console.log('test');
}
dog.set('attr', {
    age: '1 year',
    breed: 'Bernese Mountain Dog'
});
  3. Place 1 space before the opening parenthesis in control statements (if, while, etc.). Place no space before the argument list in function calls and declarations. Place one space before the argument list in anonymous function declarations. Example 
    
// bad
if(isJedi) {
    fight ();
}

// good
if (isJedi) {
    fight();
}

// bad
function fight () {
    console.log ('Swooosh!');
}

// good
function fight() {
    console.log('Swooosh!');
}

// bad
const fight = function() {
    console.log ('Swooosh!');
}

// good
const fight = function () {
    console.log ('Swooosh!');
}

// bad
const fight =()=> {
    console.log ('Swooosh!');
}

// good
const fight = () => {
    console.log ('Swooosh!');
}
  4. Set off operators with spaces. Example 
    
// bad
const x=y+5;

// good
const x = y + 5;
  5. Use indentation when making long method chains. Use a leading dot, which emphasizes that the line is a method call, not a new statement. Example 
    
// bad
$('#items').find('.selected').highlight().end().find('.open').updateCount();

// good
$('#items')
    .find('.selected').highlight().end()
    .find('.open').updateCount();
  6. Leading commas: Nope. Example 
    
// bad
const story = [
    once
  , upon
  , aTime
];

// good
const story = [
    once,
    upon,
    aTime
];
  7. Additional trailing comma: Nope. Example 
    
// bad
const hero = {
    firstName: 'Kevin',
    lastName: 'Flynn',
};

// good
const hero = {
    firstName: 'Kevin',
    lastName: 'Flynn'
};
  8. Semicolons are mandatory Example 
    
// bad
const $block = this.$element.find('.block')
$block.css('border', '1px solid #ccc')

// good
const $block = this.$element.find('.block');
$block.css('border', '1px solid #ccc');
  9. Useless parentheses around expressions should be removed to prevent any misunderstanding Example 
    
// bad
return (x);
return (x + 1);
const x = (y / 2 + 1);
const func = (n) => {
    return n * 2;
}

// good
return x;
return x + 1;
const x = y / 2 + 1;
const func = n => n * 2;
  10. Perform type coercion at the beginning of the statement Example 
    
// bad
const totalScore = this.reviewScore + '';

// good
const totalScore = '' + this.reviewScore;

// bad
const totalScore = '' + this.reviewScore + ' total score';

// good
const totalScore = this.reviewScore + ' total score';
  11. Use parseInt for Numbers and always with a radix for type casting Example 
    
const inputValue = '4';

// bad
const val = new Number(inputValue);

// bad
const val = +inputValue;

// bad
const val = inputValue >> 0;

// bad
const val = parseInt(inputValue);

// good
const val = Number(inputValue);

// good
const val = parseInt(inputValue, 10);
  12. Booleans: Example 
    
const age = 0;

// bad
const hasAge = new Boolean(age);

// good
const hasAge = Boolean(age);

// good
const hasAge = !!age;
  13. Use dot notation when accessing properties Example 
    
const luke = {
    jedi: true,
    age: 28
};

// bad
const isJedi = luke['jedi'];

// good
const isJedi = luke.jedi;
  14. Use subscript notation [] when accessing properties with a variable or a property with underscore Example 
    
const luke = {
    jedi: true,
    is_evil: false,
    age: 28
};

const getProp = prop => {
    return luke[prop];
}

const isJedi = getProp('jedi');
const isEvil = luke['is_evil'];
  15. Try to avoid comments in your code. Your code should be readable without comments! Delete comments wherever it possible!
  16. Use only /** ... */ for multi-line comments. Don't use // for multi-line comments! Example 
    
// bad
// make() returns a new element
// based on the passed in tag name

// good
/**
 * make() returns a new element
 * based on the passed in tag name
 */
  17. Use // for single line comments. Use a new line for every comment. Don't use inline comments! Example 
    
// bad
const active = true;  // is current tab

//good

// is current tab
const active = true;

const getType = () => {
    // set the default type to 'no type'
    var type = this._type || 'no type';
    return type;
}
