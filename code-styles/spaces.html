Spaces and code formating
Best practices Variables Declarations Functions Declarations Conditional Blocks Classes and Modules
  1. Use 4 spaces instead of tabs. Configure your IDE to always use 4 spaces in every language. PhpStorm uses 2 tabs in some languages (CSS, Less, JSON), make sure that you've set 4 spaces for that languages (Preferences - Editor - Code Style).
  2. Place 1 space before the leading brace Example 
    
// bad
function test(){
    console.log('test');
}
dog.set('attr',{
    age: '1 year',
    breed: 'Bernese Mountain Dog'
});

// good
function test() {
    console.log('test');
}
dog.set('attr', {
    age: '1 year',
    breed: 'Bernese Mountain Dog'
});
  3. Place 1 space before the opening parenthesis in control statements (if, while, etc.). Place no space before the argument list in function calls and declarations. Place one space before the argument list in anonymous function declarations. Example 
    
// bad
if(isJedi) {
    fight ();
}

// good
if (isJedi) {
    fight();
}

// bad
function fight () {
    console.log ('Swooosh!');
}

// good
function fight() {
    console.log('Swooosh!');
}

// bad
var fight = function() {
    console.log ('Swooosh!');
}

// good
var fight = function () {
    console.log ('Swooosh!');
}
  4. Set off operators with spaces. Example 
    
// bad
var x=y+5;

// good
var x = y + 5;
