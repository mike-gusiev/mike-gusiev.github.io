Functions Declarations

  1. Function names should start with a lower case letter and use camel case notation. (eg: getData, render, update, initProperties)
  2. Function names exclude callbacks and event handlers should be actions (eg: getData, render, update, initProperties)
  3. Function arguments should be separated by spaces Example 
    
//bad
fun: (a,b,c) => {
    //...
}

//good
fun: (a, b, c) => {
    //...
}
  4. Assignments should not be made from within sub-expressions Example 
    
//bad
doSomething(i = 42);

//good
let i = 42;
doSomething(i);
  5. Functions should not have too many lines. Maximum authorized lines in a function - 100!
  6. Functions should not have too many parameters. The maximum authorized number of parameters - 7!
  7. Function calls should not pass extra arguments Example 
    
//bad
const say = (a, b) => {
    print(a + ' ' + b);
}
say('hello', 'world', '!');

//good
const say = (a, b) => {
    print(a + ' ' + b);
}
say('hello', 'world');
  8. Results of operations on strings should not be ignored. Every variable should be used or returned as result. Example 
    
//bad
fun: () => {
    let str = "...";
    str.toUpperCase();
}

//good
fun: () => {
    let str = '...';
    str = str.toUpperCase();
    return str;
}
  9. Each function should be separated by empty line.
  10. IIFEs are forbidden. Use blocks instead Example 
    
// bad. Immediately Invoked Function Expressions are obsolete!
(function () {
    var food = 'Meow Mix';
    // ...
}());

// good
{
    let food = 'Meow Mix';
    // ...
}
  11. Use arrow functions to preserve "this" value Example 
    
// bad
setTimeout(function () {
    console.log(this.value);
}.bind(this), 1000);

// bad
var that = this;
setTimeout(function () {
    console.log(that.value);
}, 1000);

// good
setTimeout(() => {
    console.log(this.value);
}, 1000);
  12. Use promises instead of nested code Example 
    
// bad
func1(function (value1) {
    func2(value1, function (value2) {
        func3(value2, function (value3) {
            func4(value3, function (value4) {
                func5(value4, function (value5) {
                    // Do something with value 5
                });
            });
        });
    });
});

// good
func1(value1)
    .then(func2)
    .then(func3)
    .then(func4)
    .then(func5, value5 => {
        // Do something with value 5
    });
  13. Use function default parameters instead of chain assignments Example 
    
// bad
function addTwoNumbers(x, y) {
    x = x || 0;
    y = y || 0;
    return x + y;
}

// good
const addTwoNumbers = (x = 0, y = 0) => x + y;
  14. Use rest operator "..." instead of "arguments" Example 
    
// bad
function logArguments(par1) {
    console.log(par1);
    for (var i=1; i < arguments.length; i++) {
        console.log(arguments[i]);
    }
}

// good
const logArguments = (par1, ...args) => {
    console.log(par1);
    for (let arg of args) {
        console.log(arg);
    }
};
  15. Use spread operator whenever it possible Example 
    
// bad. In ES5, we could find the max of values in an array by using the apply method on Math.max like this:
Math.max.apply(null, [-1, 100, 9001, -32]); // 9001

// good. In ES6, we can now use the spread operator to pass an array of values to be used as parameters to a function:
Math.max(...[-1, 100, 9001, -32]); // 9001

// good. array concatenation
let cities = ['San Francisco', 'Los Angeles'];
let places = ['Miami', ...cities, 'Chicago']; // ['Miami', 'San Francisco', 'Los Angeles', 'Chicago']
