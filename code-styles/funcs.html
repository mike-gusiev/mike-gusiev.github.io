Functions Declarations
Best practices Spaces and code formating Variables Declarations Conditional Blocks Classes and Modules
  1. Function names should start with a lower case letter and use camel case notation. (eg: getData, render, update, initProperties)
  2. Function names exclude callbacks and event handlers should be actions (eg: getData, render, update, initProperties)
  3. Function arguments should be separated by spaces Example 
    
//bad
fun: (a,b,c) => {
    //...
}
    
//good
fun: (a, b, c) => {
    //...
}
  4. Assignments should not be made from within sub-expressions Example 
    
//bad
doSomething(i = 42);
    
//good
let i = 42;
doSomething(i);
  5. Functions should not have too many lines. Maximum authorized lines in a function - 100!
  6. Functions should not have too many parameters. The maximum authorized number of parameters - 7!
  7. Function calls should not pass extra arguments Example 
    
//bad
const say = (a, b) => {
    print(a + ' ' + b);
}
say('hello', 'world', '!');
    
//good
const say = (a, b) => {
    print(a + ' ' + b);
}
say('hello', 'world');
  8. Results of operations on strings should not be ignored. Every variable should be used or returned as result. Example 
    
//bad
fun: () => {
    let str = "...";
    str.toUpperCase();
}
    
//good
fun: () => {
    let str = '...';
    str = str.toUpperCase();
    return str;
}
  9. Each function should be separated by empty line.
